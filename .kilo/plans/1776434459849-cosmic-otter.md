# Development Environment Setup Plan — Upskiill

## Overview
This document provides step-by-step instructions for installing all required tools and dependencies to run the Upskiill platform locally on Windows and macOS.

---

## Project Structure
```
upskiill/
├── frontend/      # Next.js 14 + React + TypeScript (port 3000)
├── backend/       # NestJS + TypeScript (port 3001)
└── docs/          # Documentation
```

| Tool | Purpose | Recommended Version |
|------|---------|---------------------|

## Installation Instructions

### 1. Node.js >= 18

#### Windows
1. Download from [nodejs.org](https://nodejs.org) (LTS version)
2. Run the installer — check "Add to PATH"
3. Verify:
```powershell
node --version
npm --version
```

#### macOS
Using Homebrew (recommended):
```bash
brew install node@18
brew link --force --overwrite node@18
```

Or download from [nodejs.org](https://nodejs.org)

Verify:
```bash
node --version
npm --version
```

---

### 2. Package Manager (choose one)

**npm** — included with Node.js (default)

**pnpm** (optional, faster):
```bash
npm install -g pnpm
```

**yarn** (optional):
```bash
npm install -g yarn
```

---

### 3. Git

#### Windows
- Download from [git-scm.com](https://git-scm.com)
- Run installer with default options

Verify:
```powershell
git --version
```

#### macOS
```bash
brew install git
```

Verify:
```bash
git --version
```

---

### 4. PostgreSQL Database

#### Windows

**Option A: Using Installer (most straightforward)**
1. Download from [postgresql.org/download](https://www.postgresql.org/download/windows/)
2. Use the interactive installer by EnterpriseDB
3. Set a password for the `postgres` superuser (remember it!)
4. Keep default port `5432`
5. After install, open "pgAdmin 4" (GUI) or use command line

**Option B: Using WSL2 (if available)**
```bash
wsl
sudo apt update && sudo apt install postgresql postgresql-contrib
```

**Create the database:**
```powershell
# Open PostgreSQL command line as postgres user
psql -U postgres
```
Then run:
```sql
CREATE DATABASE upskiill;
CREATE USER upskiill_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE upskiill TO upskiill_user;
\q
```

#### macOS

**Using Homebrew:**
```bash
brew install postgresql@15
brew services start postgresql@15
```

**Create the database:**
```bash
# Create database and user
createdb upskiill
# Or via psql:
psql postgres
```
```sql
CREATE DATABASE upskiill;
CREATE USER upskiill_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE upskiill TO upskiill_user;
\q
```

**Verify PostgreSQL is running:**
```bash
pg_isready
```

---

### 5. Redis Cache (optional but recommended)

#### Windows
1. Download from [redis.io/download](https://redis.io/download) — look for "Microsoft Open Tech" Redis for Windows
2. Or use WSL2: `sudo apt install redis-server`
3. Start Redis server (it should run as a service)

Alternative: Use Docker (see below)

#### macOS
```bash
brew install redis
brew services start redis
```

Verify:
```bash
redis-cli ping
# Should return: PONG
```

---

### 6. Docker (optional — simplifies database & Redis setup)

If you prefer containerized services:

#### Windows & macOS
1. Download from [docker.com](https://www.docker.com/products/docker-desktop)
2. Install Docker Desktop
3. Create `docker-compose.yml` in project root:
```yaml
version: '3.8'
services:
  postgres:
    image: postgres:15-alpine
    restart: always
    environment:
      POSTGRES_DB: upskiill
      POSTGRES_USER: upskiill_user
      POSTGRES_PASSWORD: your_password
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine
    restart: always
    ports:
      - "6379:6379"

volumes:
  postgres_data:
```

Then:
```bash
docker-compose up -d
```

## Project Setup

### Clone Repository (if not already done)
```bash
git clone https://github.com/upskiill201/upskiill.git
cd upskiill
```

---

### Backend Setup (NestJS)

1. Navigate to backend folder:
```bash
cd backend
```

2. Install dependencies:
```bash
npm install
```

3. Create `.env` file:
```bash
# Copy from example if exists, or create new
cp .env.example .env  # (if .env.example exists)
```
Create `backend/.env` with:
```env
# Database
DATABASE_URL="postgresql://upskiill_user:your_password@localhost:5432/upskiill?schema=public"
DIRECT_URL="postgresql://upskiill_user:your_password@localhost:5432/upskiill?schema=public"

# JWT
JWT_SECRET="your-super-secret-jwt-key-change-this"
JWT_EXPIRY="7d"

# Firebase (get from Firebase Console)
FIREBASE_ADMIN_KEY='{"type":"service_account",...}'
FIREBASE_PROJECT_ID="your-project-id"

# Stripe (get from Stripe Dashboard)
STRIPE_SECRET_KEY="sk_test_..."
STRIPE_WEBHOOK_SECRET="whsec_..."

# Paystack (optional)
PAYSTACK_SECRET_KEY="sk_live_..."

# Email (SendGrid)
SENDGRID_API_KEY="SG...."

# Redis (optional)
REDIS_HOST="localhost"
REDIS_PORT=6379

# AWS (for video storage — optional for local)
AWS_ACCESS_KEY_ID="your_key"
AWS_SECRET_ACCESS_KEY="your_secret"
AWS_REGION="us-east-1"
AWS_S3_BUCKET="upskiill-videos"
```

4. Generate Prisma client & run migrations:
```bash
npx prisma generate
npx prisma migrate dev --name init
# Optional: seed database
npx prisma db seed
```

5. Start backend in development mode:
```bash
npm run start:dev
```
Backend runs at: **http://localhost:3001**

---

### Frontend Setup (Next.js)

1. Navigate to frontend folder:
```bash
cd frontend
```

2. Install dependencies:
```bash
npm install
```

3. Create `.env.local` file:
```bash
# Create file
echo "NEXT_PUBLIC_API_URL=http://localhost:3001" > .env.local

```
```env
NEXT_PUBLIC_API_URL=http://localhost:3001
NEXT_PUBLIC_APP_URL=http://localhost:3000
# Firebase config (get from Firebase Console)
NEXT_PUBLIC_FIREBASE_API_KEY="..."
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN="..."
NEXT_PUBLIC_FIREBASE_PROJECT_ID="..."
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET="..."
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID="..."
NEXT_PUBLIC_FIREBASE_APP_ID="..."

# Stripe publishable key
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY="pk_test_..."
```

4. Start frontend development server:
```bash
npm run dev
```
Frontend runs at: **http://localhost:3000**

## Common Issues & Troubleshooting

### Port Already in Use
If ports 3000 or 3001 are busy:
- Find process: `netstat -ano | findstr :3000` (Windows) or `lsof -i :3000` (macOS)
- Kill it: `taskkill /PID <pid> /F` (Windows) or `kill -9 <pid>` (macOS)

### PostgreSQL Connection Refused
- Ensure PostgreSQL service is running:
  - Windows: Check Services app for `postgresql-x64-15`
  - macOS: `brew services list` | `brew services start postgresql@15`
- Verify `DATABASE_URL` in `.env` matches your DB credentials
- Check that database was created (`createdb upskiill`)

### Prisma Client Not Found
```bash
cd backend
npx prisma generate
```

### Module Not Found Errors
```bash
# Delete node_modules and reinstall
rm -rf node_modules package-lock.json
npm install
```

### Firebase Config Missing
Frontend requires Firebase config in `.env.local`. Get from Firebase Console:
1. Go to Project Settings → General → Your apps
2. Copy config object into env vars
