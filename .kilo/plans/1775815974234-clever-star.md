## Waitlist Strategy - Separate Repo Approach

The waitlist will be a **separate Next.js project** in a different GitHub repo.

### Deployment Flow
```
BEFORE LAUNCH:
Teyro.app → Points to waitlist project (separate GitHub repo, separate Vercel deploy)
upskiill.vercel.app → Main app continues development (for testing)

AT LAUNCH:
Teyro.app → Change to point to main project → LIVE!
```

### Benefits
- ✅ Zero risk of blocking development
- ✅ Can continue building main app locally and on upskiill.vercel.app
- ✅ Clean separation - just switch domain pointer when ready
- ✅ No middleware complexity
